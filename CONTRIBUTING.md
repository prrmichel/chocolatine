# Contributing to Chocolatine

Thanks for your interest in contributing.

## Project posture

Chocolatine is maintained on a **best-effort** basis. Contributions are welcome, but review and merge timelines are not guaranteed.

## Before opening a pull request

1. Open or link a GitHub issue describing the change.
2. Keep the PR focused on one concern.
3. Run the local publication gate:
   ```bash
   npm run gate:publication-local
   ```
4. Include short notes about what changed and why.

## Pull request expectations

- Prefer minimal, surgical changes over broad refactors.
- Update related docs when behavior or setup changes.
- Keep naming and messaging aligned with Chocolatine public posture.

## Issue intake

- Use the provided issue templates for bugs, docs, and feature requests.
- Feature requests are welcome but non-committal and best-effort.
- For vulnerabilities, do **not** open a public issue. Use [SECURITY.md](SECURITY.md).

## Community standards

By participating, you agree to follow the [Code of Conduct](CODE_OF_CONDUCT.md) and [Support](SUPPORT.md) policies.
