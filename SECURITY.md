# Security Policy

## Reporting a Vulnerability

Please report vulnerabilities privately through GitHub Security Advisories:

- Repository advisory form: `https://github.com/prrmichel/chocolatine/security/advisories/new`

Do **not** include exploit details in public issues, pull requests, or discussions.

## What to include

- A clear description of the issue and affected component.
- Reproduction steps or proof-of-concept details.
- Impact assessment (what an attacker could do).
- Suggested mitigation (if known).

## Disclosure process

- Reports are triaged on a best-effort basis.
- If confirmed, fixes are prepared before public disclosure whenever practical.
- Public disclosure should happen only after a fix or mitigation path is available.

## Scope notes

For this project, credential handling and local persistence are in scope, including Azure DevOps PAT storage behavior.
